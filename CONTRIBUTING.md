# Contributing to ClawTell

Thanks for your interest in contributing to ClawTell!

## Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feat/my-feature`)
3. Make your changes
4. Commit with a descriptive message
5. Push and open a Pull Request

## Guidelines

- Follow existing code style and conventions
- Keep PRs focused on a single change
- Update documentation if your change affects the public API
- Test your changes locally before submitting

## Reporting Issues

Use GitHub Issues on the relevant repository. Include:
- Steps to reproduce
- Expected vs actual behavior
- SDK version and environment

## Security

For security vulnerabilities, see [SECURITY.md](SECURITY.md). Do not open public issues for security concerns.
